0. 기본 설계 원칙 (가장 중요)

이 페이지는 스크롤 인터랙션이 핵심이므로
“스크롤을 누가 소비하는지”를 명확히 분리해서 설계한다.

모든 섹션은 역할이 명확히 구분된 독립 구조로 작성한다.

가로 스크롤 연출은 실제 가로 스크롤이 아닌, 세로 스크롤을 가로 이동으로 치환하는 방식으로만 구현한다.

스크롤 기준은 단 하나만 사용하며, 혼합 기준(vh, vw, scrollWidth, magic number)을 사용하지 않는다.

🔷 1. 레이아웃 기본 조건

가로 사이즈는 vw 단위를 절대 사용하지 않는다

모든 가로 크기는 width: 100% 기준으로만 설정한다.

이유: 세로 스크롤바 영역으로 인한 미세 가로 스크롤 방지.

overflow-x 문제를 숨김으로 해결하지 말고,
레이아웃이 본질적으로 넘치지 않게 설계한다.

🔷 2. 페이지 전체 구성 (총 3개 섹션)

페이지는 아래 3개 섹션만 존재해야 한다.

Sub Hero Section

Slogan Section

Equipment Section

이 외의 중간 섹션, 더미 요소, spacer, 보정용 div 생성 금지.

🔷 3. Sub Hero Section 설계 요구사항
3-1. 기본 역할

페이지 진입 시 좌 → 우로 열리는 인트로 모션으로 등장한다.

이 인트로 모션은 다른 메뉴 페이지의 서브히어로와 완전히 동일한 방식으로 동작해야 한다.

이 섹션은 수평 스크롤 연출의 시작점이다.

3-2. 디자인 조건

하단에는 파도 형태의 곡선 디테일이 반드시 포함된다.

이 파도는 장식 요소이며, 스크롤 계산에 영향을 주지 않는다.

🔷 4. Sub Hero → Slogan 전환 (가로 스크롤 핵심)
4-1. 전환 방식

휠 다운 시:

Sub Hero와 Slogan 섹션이 하나의 수평 트랙처럼 동작

화면을 좌측으로 밀어내는 가로 이동 효과 발생

실제 가로 스크롤은 사용하지 않는다.

세로 휠 입력을 가로 이동으로 1:1 치환한다.

4-2. 구조 원칙

가로 이동을 담당하는 요소와

화면을 고정(pin / sticky)하는 요소는 역할이 분리되어야 한다.

pin 해제 시 화면이 비는 구간이 절대 발생하면 안 된다.

🔷 5. Slogan Section 상세 요구사항
5-1. 구조

Slogan Section은 두 개의 본문 블록으로 구성된다.

Slogan Text 01

Slogan Text 02

두 본문은 동시에 보이지 않는다.

5-2. 등장 타이밍 규칙 (정확히 지킬 것)

Slogan Section이 화면의 85% 이상 진입했을 때

Text 01이 fade-down 애니메이션으로 등장

스크롤을 더 내리면:

Text 01은 fade-out

Text 02는 fade-down 등장

이 모든 애니메이션은:

스크롤 진행도 기반

타이밍 하드코딩 금지

5-3. 스크롤 종료 조건

Text 02 상태에서 휠 다운 시:

수평 스크롤은 완전히 종료

이후 자연스럽게 세로 스크롤로 전환

다음 섹션(Equipment)으로 이동

🔷 6. Equipment Section (장비 소개)

이 섹션부터는 일반적인 세로 스크롤 섹션

수평 스크롤, pin, sticky 효과는 절대 적용하지 않는다

앞선 인터랙션과 충돌하지 않도록 완전히 독립된 흐름으로 설계한다.

🔷 7. 스크롤 리버스 규칙 (매우 중요)

모든 스크롤 인터랙션은 휠 업 시 완전히 역재생(reverse) 되어야 한다.

상태 스킵, 점프, 순간 이동 금지.

다음을 반드시 만족해야 한다:

장비소개 → 슬로건2 → 슬로건1 → 서브히어로

순서가 깨지지 않을 것

중간에 빈 화면이 절대 나타나지 않을 것

🔷 8. 안정성 및 금지 사항

Lenis, 커스텀 스크롤 라이브러리 사용 금지

vw, 100vh 남용 금지

임의의 margin-top, -offset, 보정용 height 금지

“일단 보이게” 만드는 임시 처리 금지

구조 설명 없이 코드만 작성하는 행위 금지

🔷 9. 최종 목표

이 페이지는 다음 조건을 반드시 만족해야 한다.

가로 스크롤 오류 없음

pin 해제 시 흰 화면 없음

스크롤 싱크 어긋남 없음

구조적으로 이해 가능한 코드

다른 페이지에도 동일한 패턴으로 확장 가능한 설계

이 페이지는 “효과를 구현하는 페이지”가 아니라
“스크롤 구조를 설계하는 페이지”다.
